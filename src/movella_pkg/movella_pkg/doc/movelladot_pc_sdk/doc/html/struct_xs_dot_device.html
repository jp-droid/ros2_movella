inline

Get the info for a recording on the DOT Device with for a given index. The recording indices start at 1 and end at 254.

This function will request the information from a recording for the supplied index. If the recording does not exist or something went wrong during data transfer the recording info will be empty. The total recording time is only available for timed recordings, it will be 0xFFFF (65535) for non-timed recordings. Make sure to check the

See also
lastResult() when the XsRecordingInfo object is empty (all zeroes).
Note
Device has to be in the Idle state for this function to work.
Parameters
recordingIndexIndex of the recording to get information on.
Returns
XsRecordingInfo object with the recording information from the device, empty XsRecordingInfo if the recording does not exist or when retrieving the data from the device failed..

◆ getRecordingTime()

XsRecordingTimeInfo XsDotDevicePtr::getRecordingTime ( void  )
inline

Get the recording time info from the device. Contains startUTC, total recording time and remaining recording time.

Device has to be in the Recording state for this function to work. startUTC is the recording start UTC time in second. Total recording time is the time of a timed recording in seconds. 0xFFFF for normal (non-timed) recording. RemainingRecordingTime is the remaining time of a timed recording in seconds. 0xFFFF for normal (non-timed) recording.

Returns
XsRecordingTimeInfo with the info if request was successful. Otherwise an XsRecordingInfo with startUTC = 0 and total and remaining recording time of 0xFFFF.

◆ startExportRecording()

bool XsDotDevicePtr::startExportRecording ( int  recordingIndex)
inline

Start to download a recording and output over the onRecordedDataAvailable() callback.

Initiate a download from the device of a file. While the download is running, the device will emit the data from the recording over the onRecordedDataAvailable() callback. Even though the recorded data will come in at the maximum transfer speed (not real time), be aware that this process may take a long time depending on the size of the recording. While the export is running, the device will occasionally emit the onProgressUpdated() callback if this information is available. This function itself will return before the process is complete.

Before calling this function you should call selectExportData() to select the fields that you want exported. If you want to log the recording directly to a CSV file, you should also call enableLogging() beforehand and disableLogging() afterwards.

Note
Device has to be in the Idle state for this function to work.
Progress updates will only be emitted when the export data contains the Timestamp, and progress updates will only be representative for timed recordings.
Exporting via USB is the recommended way of exporting as it is significantly faster than exorting via BLE.
Parameters
recordingIndexIndex of the recording to download
See also
getFlashInfo().
Returns
True when the export has finished successfully, false otherwise.

rootNodeMacThe mac address of the DOT that will be the root node (master) of sync

Set the output rate and filter profile before starting the synchronization. Since the sensor will enter measurement mode right after the sync succeeds so it’s not possible to change it after sync.

Attention
This function will take at least 14 seconds and will return when all devices are synced. (refer to section 5.3.5 of the Movella DOT BLE Services Specifications)
